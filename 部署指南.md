# 部署指南 - 剧本到分镜生产流程

本项目是一个完整的前后端应用，支持多种部署方式。

## 📋 项目结构

```
├── storyboard-workflow.html      # 前端页面
├── storyboard_agent_server.py    # Python 后端服务
├── AGENTS.md                      # Agent 规则文件
├── requirements.txt               # Python 依赖
├── Dockerfile                     # Docker 配置
├── railway.json                   # Railway 配置
└── render.yaml                    # Render 配置
```

## 🚀 方案一：Railway 部署（推荐，最简单）

**优点：** 免费额度 $5/月，自动 HTTPS，部署最快
**适合：** 小规模使用，快速上线

### 步骤：

1. **注册 Railway 账号**
   - 访问 [railway.app](https://railway.app)
   - 使用 GitHub 账号登录

2. **创建新项目**
   ```bash
   # 方法 1：使用 Railway CLI（推荐）
   npm install -g @railway/cli
   railway login
   railway init
   railway up
   
   # 方法 2：通过 GitHub 仓库
   # 在 Railway 控制台选择 "New Project" → "Deploy from GitHub repo"
   ```

3. **配置环境变量（可选）**
   在 Railway 控制台添加：
   ```
   HOST=0.0.0.0
   PORT=8787
   
   # 如果要启用服务器端大模型（可选）
   STORYBOARD_LLM_ENABLED=true
   LLM_BASE_URL=https://api.openai.com/v1
   LLM_MODEL=gpt-4o-mini
   LLM_API_KEY=你的API密钥
   ```

4. **获取公网地址**
   - Railway 会自动生成一个 `.railway.app` 域名
   - 例如：`https://storyboard-workflow-production.up.railway.app`

5. **访问你的应用**
   - 直接访问生成的域名即可使用

## 🌐 方案二：Render 部署（免费层）

**优点：** 完全免费，自动 HTTPS
**缺点：** 15 分钟无访问会休眠，首次访问需等待唤醒

### 步骤：

1. **注册 Render 账号**
   - 访问 [render.com](https://render.com)
   - 使用 GitHub 账号登录

2. **创建 Web Service**
   - 点击 "New +" → "Web Service"
   - 连接你的 GitHub 仓库
   - Render 会自动检测到 `render.yaml` 配置

3. **配置（自动读取 render.yaml）**
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python storyboard_agent_server.py`
   - 选择 **Free** 计划

4. **环境变量（可选）**
   在 Render 控制台添加：
   ```
   STORYBOARD_LLM_ENABLED=true
   LLM_BASE_URL=https://api.openai.com/v1
   LLM_MODEL=gpt-4o-mini
   LLM_API_KEY=你的API密钥
   ```

5. **获取公网地址**
   - 例如：`https://storyboard-workflow.onrender.com`

## 🐳 方案三：Docker 部署（适合自有服务器）

**优点：** 完全控制，适合长期运行
**适合：** 有自己的云服务器（阿里云、腾讯云、AWS 等）

### 步骤：

1. **构建 Docker 镜像**
   ```bash
   docker build -t storyboard-workflow .
   ```

2. **运行容器**
   ```bash
   docker run -d \
     -p 8787:8787 \
     --name storyboard \
     -e HOST=0.0.0.0 \
     -e PORT=8787 \
     storyboard-workflow
   ```

3. **配置 Nginx 反向代理（推荐）**
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       
       location / {
           proxy_pass http://localhost:8787;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
       }
   }
   ```

4. **配置 SSL 证书（使用 Certbot）**
   ```bash
   sudo certbot --nginx -d your-domain.com
   ```

## 🔐 安全建议

1. **不要在前端代码中硬编码 API 密钥**
2. **使用环境变量配置服务器端 LLM**
3. **添加访问频率限制**（防止滥用）
4. **使用 HTTPS**（Railway 和 Render 自动提供）
